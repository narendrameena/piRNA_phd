SOURCE DATA FOR FIGURES — piRNA phasing (for publication / journal source-data requirements)
==============================================================================================
Generated 2026-06-07. Method for all phasing figures: reads trimmed with cutadapt 5.0
(--minimum-length 20 --maximum-length 36 --discard-untrimmed -a TGGAATTCTCGGGTGCCAAGG),
aligned with STAR 2.7.11b to each strain's own genome index, ONE RANDOM coordinate per read
(--outSAMmultNmax 1 --outMultimapperOrder Random --runRNGseed 777). Phasing computed on
24-32 nt reads as the distance from the 3' end of one piRNA to the 5' end of the next
same-strand piRNA (GenomicRanges::follow, per Almeida et al. Genome Biol 2025, PMID 39844208);
"+1 nt" = directly adjacent = phased (Zucchini-dependent) biogenesis.

COLUMN DEFINITIONS
----------------------------------------------------------------------------------------------
strain, subspecies (classical M.m.domesticus vs wild: CAST/PWK/SPRET/WSB), timepoint
(E16.5=16.5dpc, P12.5=12.5dpp, P20.5=20.5dpp), replicate (1/2/3).
plus1_pct          = % of all adjacent same-strand piRNA pairs that are at +1 nt (phased)
plus1_fraction     = same as fraction (0-1)
plus1_zscore       = z-score of the +1-nt bin vs the 1-50 nt distance distribution
count_plus1        = number of +1-nt (phased) pairs
n_adjacent_pairs   = total adjacent same-strand pairs counted
total_alignments_24_32nt = piRNA-sized alignments used
distance / pair_count / zscore (distributions file) = full 1-50 nt histogram per sample

NOTE: timepoint order in all figures is fixed developmental order E16.5 -> P12.5 -> P20.5.
Strain order in ALL figures follows the canonical figure order (thesis Figure 4.4, strains
ordered by median P20.5 PC1 from the RNA-seq PCA): C57BL_6, C57BL_6NJ, BALB_cJ, A_J, FVB_NJ,
C3H_HeJ, LP_J, 129S1_SvImJ, DBA_2J, AKR_J, CBA_J, NZO_HlLtJ, NOD_ShiLtJ, WSB_EiJ, CAST_EiJ,
PWK_PhJ, SPRET_EiJ  (defined once in strain_order.py; C57BL_6 reference has no phasing data).
NOTE: all-strains run is 143/144 samples (CBA_J-16.5dpc.1 not yet processed).
